# Heap（堆）核心笔记

## 一、基本结构

- 完全二叉树， 用数组存储 
- index 关系：

```
parent = (i - 1) // 2
left   = 2*i + 1
right  = 2*i + 2
```

---

## 二、Heap 类型

### 大根堆（Max Heap) 
- 父节点 ≥ 子节点
- 最大值在 root

---

## 三、核心操作

### 1️⃣ Heap Insert（从底到顶）, (i -1)//2

步骤：
1. 插入到数组末尾
2. 与父节点比较
3. 不满足就交换，向上调整

时间复杂度：
```
O(log N)
```

---

### 2️⃣ Heapify（从顶到底） max(2*i +1, 2*i +2 )

步骤：
1. root 与子节点比较
2. 选择更大的 child
3. 交换并向下调整

时间复杂度：
```
O(log N)
```

---

## 四、Build Heap（建堆）

方法：
```
for i = (N-2)//2 → 0:
    heapify(i)
```

时间复杂度：
```
O(N) = N/2 + N/4 + ....
```

原因：
- 底层节点多但高度低
- 总体调整成本小

---

## 五、Heap Sort

步骤：

1. 建堆
```
O(N)
```

2. 反复交换 + heapify
```
for i = N-1 → 1:
    swap(arr[0], arr[i])
    heapify(0, i)
```

总复杂度：
```
O(N log N)
```

---

## 六、方向总结

| 操作 | 方向 | 场景 |
|------|------|------|
| insert | 从底到顶 | 新元素加入 |
| heapify | 从顶到底 | 删除/调整 |

---

## 七、核心理解（面试必问）

1. 为什么是完全二叉树? 
→ 保证数组连续 + index 可计算

2. 为什么 build heap 是 O(N)？
→ 底层节点多但高度低

3. heap sort 是否稳定？
→ ❌ 不稳定
